# AI 프롬프트 변환기

사용자가 작업 유형과 핵심 조건을 선택하면, 역할·목표·맥락·입력 자료·기준·금지사항·출력 형식·검토 방식이 포함된 실행형 프롬프트를 생성하는 정적 웹 도구입니다.

## 지원 작업 유형

1. 검색
2. 코딩
3. 요약
4. 아이디어
5. 글쓰기
6. 투자/재테크
7. 마케팅
8. 리서치/분석
9. 여행/코스
10. 재미/생활/기타
11. 이미지 생성

## 주요 반영 사항

- 글쓰기 유형을 세분화했습니다.
  - 일반 글쓰기/문장 작성
  - 블로그 글/후기/정보글
  - 유튜브 스크립트
  - 짧은 소설/웹소설 플롯
  - 문장 수정/교정/윤문
  - 책 원고/챕터
  - SNS 게시글/스레드
- 여행/코스를 독립 작업 유형으로 분리했습니다.
  - 목적지/출발지, 여행 기간, 목적/컨셉, 동행자 정보는 필수값으로 받습니다.
  - 예산, 숙소/이동, 동선 강도, 필수 장소, 피해야 할 조건, 비상 대안은 보조값으로 받습니다.
  - 결과는 일정표, 동선, 예상 소요 시간, 예산표, 준비물, 현지 팁, 대안 코스 중심으로 생성됩니다.
- 재미/생활/기타에서는 여행을 제거하고 운세/타로/사주, 생활 루틴, 밈/상황극, 기타로 정리했습니다.
- 사용자가 붙여넣은 예시는 고정값으로 복사하지 않고, 프롬프트 설계 방향만 참고했습니다.
- 각 입력 필드에 값이 있을 때 해당 필드만 지우는 X 버튼을 표시합니다.
- 샘플 입력은 토글 방식입니다. 다시 누르면 샘플로 입력된 값만 지우고 사용자가 수정한 값은 유지합니다.
- 셀렉트의 권장값은 기본 셀렉트의 텍스트 일부를 CSS로 꾸미는 대신, 커스텀 셀렉트 UI에서 `권장` 딱지와 권장 사유가 분리되어 보이도록 표시됩니다.
- 원본 `<select>` 값은 유지하므로 프롬프트 생성, 검증, 기본값 적용 로직은 그대로 동작합니다.

## 파일 구조

```text
prompt-builder/
├─ index.html
├─ assets/
│  ├─ app.js
│  └─ styles.css
├─ data/
│  └─ prompt-config.js
├─ README.md
└─ IMPLEMENTATION_AUDIT.md
```

## 실행 방법

외부 라이브러리가 없으므로 `index.html`을 바로 열어도 됩니다. 로컬 서버로 확인하려면 아래 중 하나를 사용하세요.

```bash
npx serve prompt-builder
```

또는 VS Code Live Server로 `index.html`을 실행하면 됩니다.

- 조건부 필수 영역에 별도 카드 스타일과 필수 배지를 적용했습니다.
- 커스텀 셀렉트 권장 배지 HTML 구조를 안정화했습니다.
