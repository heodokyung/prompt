생성 규칙

  • 모든 프롬프트는 역할, 목표, 맥락, 입력 자료, 기준, 금지사항, 검토 방식을 포함합니다.
  • 각 유형마다 서로 다른 인격체가 의견을 내고 최종 결정권자가 정리합니다.
  • 투자/재테크, 마케팅, 리서치/분석, 재미/생활/기타 유형은 별도 역할과 출력 형식을 사용합니다.
  • 샘플 입력은 토글 방식입니다. 다시 누르면 샘플값만 지웁니다.
  • 거짓 정보, 근거 없는 단정, 편견에 의존하지 않도록 방어 문구를 포함합니다.
  • 내부 추론 전문 대신 핵심 근거와 판단 단계만 정리하도록 요청합니다.
